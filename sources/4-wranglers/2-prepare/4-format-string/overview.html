Format String

Normalize, Convert Case, Combine, Split, Truncate, Detect, Remove non-numeric, and Remove non-alpha-numeric

Single text characters like the alphabet (A-Z), numeral digits (0-9), punctuation (. ! ? ; :), single/double quote marks (’ ") form a string-type data. Therefore you will encounter many, if not most, data sources which contain string-type data. The text characters can be combined in ways that are misleading and illegible. In order to make it easier to read, write and maintain a string-type data, formatting it properly is necessary for further analysis.

  • Normalize: Some Unicode characters have the same appearance but multiple representations. For example, “â” can be represented as one code point for “â”, or two decomposed code points for “a” and “^”. Normalize method will convert strings to NFC, NFKC, NFD, NFKD, or NFKC_Casefold Unicode Normalization Form or check whether strings are normalized. Basically, Unicode normalization is the decomposition or composition of characters.
  • Convert Case: convert the string into opposite case.
  • Combine: connect multiple strings with separators.
  • Split: break a string into multiple subsets.
  • Truncate: remove characters based on its position.
  • Detect: detect patterns, similar to wrangler-find methods.
  • Remove non-numeric: remove digits [0-9].
  • Remove non-alpha-numeric: remove alphanumeric characters [A-z0-9].
